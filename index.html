Recap of level 1

  • HTML is a set of standard tags that you can use to create elements following a set of rules inherited from a generic language structure called XML. It's like a vocabulary to XML's grammar.
  • Elements are semantic, they convey meaning without visual style
  • You can nest elements inside of one another, but it's important that they are closed in the right order so that they are well formed
  • Some tags need to be opened and closed, others are 'self-closing'
  • Attributes can be used to add further information to elements which is not displayed

Loading CSS and JavaScript

CSS:

<link rel='stylesheet' href='http://...'>

JavaScript:

<script src='http://...'></script>

Where do they go?

<html>
  <head>
    <link rel='stylesheet' href='my-stylesheet.css'>
  </head>
  <body>
    <p>Hello!</p>
    <script src='my-javascript.js'></script>
  </body>
</html>

Where do they go?

CSS goes at the top of the document, before the content. Otherwise, you can get a Flash of Unstyled Content (FOUC):

Where do they go?

JavaScript goes at the bottom of the document, after the content. It's not needed urgently and shouldn't slow down page loading.

Annoying differences

  • The <link> tag is self-closing, so you don't need a </link>, whereas the <script> tag is not, so you do need a </script>.
  • Since the <link> tag is also used to link to things other than stylesheets, you need an attribute (rel) to say that it's a stylesheet you're linking to, but you don't need that for <script>
  • The URL of the stylesheet is in an attribute called href, whilst the URL of the script is in an attribute called src.

    Recap

    • HTML doesn't give us any visual style or clever whizzy behaviours. For that we need CSS and JavaScript.
    • CSS works by linking style rules with HTML elements that are labelled with classes.
    • Separating CSS and JavaScript away from HTML means you can focus on content and leave other people to worry about style and behaviours.
    • The Origami registry is the place to go to see what clever components we have available to make your HTML work harder.
